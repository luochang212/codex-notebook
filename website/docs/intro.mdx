这不是一份线性的源码阅读记录，而是一组围绕 Codex 的研究型问题卡片。

## 研究对象

本笔记只针对 `openai/codex` 这个代码库展开。章节里的架构判断、源码切片和伪代码都应回到该仓库验证；如果讨论其他 Codex 产品形态或外部实现，必须在正文里明确说明边界。

每个条目都会保留原始问题，再把它扩展成一个可继续深入的研究主题：它涉及哪些架构层级，当前代码给出的判断是什么，以及哪些直觉需要修正。

## 为什么用这个形式

Codex 的架构问题往往不是“这个模块做什么”，而是“为什么要这样切分复杂度”。问答笔记的目标是把这些问题保留下来，让每次追问都能成为后续研究的索引。

## 记录格式

问答章节不强制套用固定模板，但通常会包含这些信息：

- **原始问题**：保留提问时的原话，避免后续解释稀释问题本身。
- **研究主题或核心判断**：把问题抽象成一个可验证的架构判断。
- **关键源码切片**：直接呈现能支撑判断的源码或注明出处的伪代码。
- **架构边界和取舍**：指出问题里容易混在一起的层级，必要时纠正错误前提。

从 [Codex 架构问答](./qa) 开始阅读。每个问答都是独立章节，先讲清楚架构判断，再直接呈现关键源码证据。
